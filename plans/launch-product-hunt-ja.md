## 0. ポジショニング — 育てるアシスタント。「収穫」が先、「種」が後（最初に読むこと）

**ビジョンを一段落で（原典より）:** 誰もが同じアシスタントを思い描いている — 自分の仕事も予定もクライアントも習慣も知っていて、24時間そばにいる存在。それはどこにも売っていない。デバイス（グラス、スマホ、ウォッチ）は入り口にすぎず、モデル（GPT、Claude）はエンジンにすぎない。アシスタントの価値は**その人についてどれだけ知っているか**で決まる — 会話、予定表、住所録、メモ、そして使っているアプリまで。AI 各社はそれをよく分かっているからこそ、こぞって「記憶」機能を積み上げている — そして、これほど重要なものを特定の一社に預けるべきではないというのが、まさにその理由である。**持つ価値のあるアシスタントは、自分のマシンの上で自分で育てたものだけ。** MulmoClaude はその畑（はたけ）— インストール直後は空で、会話によって耕されていく。

**すべての「3秒サーフェス」— 見出し、ヒーロー動画の冒頭、ツイート#1 — では、「これは買えない」という主張を、育ったアシスタントが新品のチャットボットには絶対できないことをやる姿で証明し、そのあとで種がいかに安かったかを明かす:**

> 3週間育てたアシスタントが *「今四半期のクライアント会食の支出はいくら？ 金曜のお店も予約して」* に答える — **持ち主が頼んだときに自分で作った**請求書トラッカーとレストランリストを、数週間の時を隔てて横断しながら。そこからスマッシュカットで Day 1 へ: 空のフォルダ、一文 — *「請求書トラッカーを作って」* — そしてアプリが現れる。

これがフックであり、**順序が固定された2つのビート: 収穫 → 種**で構成される。収穫（明らかに*あなたを知っている*アシスタント）はお金で買えないもの — スクロールを止めるのはこれ。種（ただ頼んだだけでそこに到達した。コードなし、アプリごとのセットアップなし）は手が届く感覚 — 試させるのはこれ。順序を逆にする（「見て、アプリが作れる！」）と、意図的に離脱したアプリビルダーの激戦区に逆戻りする。**動くサーフェス（ヒーロー動画、15秒ループ）では、最初の15秒以内に両方のビートを着地させること。** 抽象語（「メモリアーキテクチャ」「スキーマ」「DSL」「ハーネス」「ローカルファースト」）で開かない — 育ったアシスタントを見せる。理屈は後。

**中心にあるただ一つのもの: アシスタントそのもの、そして「自分で育てる」という事実。** 他のすべてはそこから放射される:

- **会話で育つ。** アプリを頼む — レストランリスト、請求書トラッカー、単語練習 — と現れる。コードなし、プラグインなし、コードを意識することすらなし。（プログラマが AI にコードを書かせるのは「バイブコーディング」。ここではコードが見えることすらない。）どれも一人のための小さなアプリ — AI がソフトウェア開発コストを限りなくゼロに近づけたからこそ、初めて経済的に成立した世界。
- **学んだことはすべて残る。** 会話はリンクされた wiki になり、構造化データは自作アプリの中に住み、セッション間で何も蒸発しない。1週間で*あなたの*文脈から答えるようになり、1か月でそれを横断して合成するようになる。
- **どこからでも同じアシスタント。** スマホと PC から同じ ID でログインすれば同じアシスタントにつながる — 中継サーバーはメッセージを転送するだけで、データもアプリも一切保存しない。将来はグラスやヘッドセットも。アシスタントの家はあなたのマシンで、デバイスは入り口。
- **寝ている間も働く。** スケジュールされたソース、朝のブリーフィング、期日前のリマインド — 畑は訪れの合間にも育つ。
- **誰にも奪えない。** プレーンな Markdown/JSON が1つのローカルフォルダに。git フレンドリー、オープンソース（MIT）。ロックイン拒否は「4番目の機能」ではなく、**プロジェクトの存在理由**として早く・はっきり述べる。

**このフレーミングが生む正直な緊張（隠さず名指しする）:** PH は即時の満足に報い、育成の見返りは数週間後に来る。「インストール直後は空の畑」は真実であると同時にアンチデモである。**解決策、優先順:** (1) デモは必ず*育った後*の状態をタイムラプスで見せる — 空の畑だけを見せない。(2) アクティベーション経路（§10.5）は「30日育ったアシスタントの見学ツアー」であって、まっさらなインストールではない。(3) オンボーディングは **「最初の種は5分」** を約束する — 一つ頼み、一つのアプリ、一つの記憶 — Day 1 にも独自の見返りを。

**緊張その2 — ビジョンはコンシューマ向け、チャネルは開発者向け。** ビジョンは万人に語りかける（「プログラミングの知識は要らない」）が、当日の投票者は Claude パワーユーザーとセルフホスト勢。**解決策:** ビジョンのロックイン批判の半分は、エンジニアにとって*母国語*である — 「AI 各社があなたの記憶の所有権を争っている。あなたのはあなたのディスクにあるべきだ」は、どんな機能よりも HN / r/selfhosted に刺さる。エンジニア向けには *「AI 各社が貸し出したがっているメモリ層をセルフホストする — そしてその中でエージェントに自分の道具を育てさせる」* とフレーミングする。コンシューマ面は温かく（「自分で育てる」）、開発者面は主権（sovereignty）で先制する。第2フェーズの読者（生産性、ナレッジワーカー、エンタープライズ）は後から来る。**JP はもはや第2フェーズではない** — ビジョン原典は日本語のナレーション台本であり、Satoshi の JP オーディエンスは強い。JP は独立した一級のビートを持つ（§8）。

**2026年の現実確認:** アプリ生成、リッチ出力、モバイル AI、サンドボックスは**コモディティ** — どれも先頭に置かない。「頼む → アプリが現れる」ですら*デモとしては*もはや当たり前（Lovable/Bolt/V0 も一応やる）。彼らが言えないのは *「そして3週間後、それはあなたを知るアシスタントの一部になっている」* — 成長の一句を必ず添える。

### ポジショニングの賭け — 「育てるアシスタント」。「AI アプリビルダー」でも「AI アシスタント」でもない

隣接カテゴリはどちらも激戦区: **「AI アプリビルダー」**（Lovable、Bolt、V0、Replit）と**「AI アシスタント」**（Siri、Alexa、Copilot、ChatGPT）。空白地帯はビジョンが特定した*接続*にある: **蓄積された個人文脈から価値が生まれ、持ち主自身が育て、持ち主が支配するハードウェアの上に住むアシスタント。** 既存勢は追随できない — 彼らのビジネスモデルこそが「蓄積を自社サーバーで起こすこと」だからである。

> AI アシスタントの価値はモデルの IQ ではない — モデルは借り物のエンジンで、誰にとっても同じ。価値は**あなたについてどれだけ知っているか**であり、それこそが他人の壁の内側で築いてはならない資産である。家で育てること自体が製品。

**展開の仕方:**
- **PH / コンシューマ面:** 温かい語彙 — *grow, nurture, yours, it knows you*。記憶競争への批判はメーカー投稿に一度だけ、動機として登場 — 十字軍にしない。
- **HN / 開発者 / セルフホスト面:** 主権で先制 — *「メモリ層をセルフホストせよ」*、記憶競争批判のフルバージョン、プレーンテキストのファイル、中継は何も保存しない設計。DSL-as-harness 論（`docs/papers/dsl-as-harness.md`）はここに住む — コメントとブログ。**PH には絶対に出さない。**
- **誇大化しない:** AGI 執事を出荷するわけではない。主張は狭く、真実である — *記憶・データ・自作アプリをあなたが所有するプレーンファイルとして蓄積し、どのデバイスからも届くアシスタントの家。*

### サブタグライン（ソーシャル + ヒーロー画像の A/B 用）

## 2. ワンセンテンス・ピッチ

**MulmoClaude は、自分のコンピュータの上でパーソナル AI アシスタントを育てるオープンソースの家である — 普通の会話で頼んだアプリを（コードなしで）作り、wiki とアプリ自身のデータにすべてを記憶し、留守中もスケジュールで働き、何も保存しない中継サーバー経由でスマホからも届く — そのすべてが、どの会社にも奪えない1つのフォルダのプレーンファイルとして。**

閲覧者が尋ねる順に節を並べている: *それは何? → どう育つ? → 残るのか? → 外から届くか? → 誰のものか?* すべての節の中心はアシスタントであり、各メカニズムはこれらの問いへの答えとしてのみ登場する。

### アンチラッパー・ライン（「ただの ChatGPT クローンでは?」が出たら必ずこれ）

> **This doesn't call the Claude API. It runs Claude Code directly — your auth, your tools, your files, your environment. The host contains zero domain code; every app is something you or the assistant grew.**
> *（意訳: Claude API を呼んでいるのではない。Claude Code そのものを直接動かしている — あなたの認証、あなたのツール、あなたのファイル、あなたの環境で。ホストにはドメインコードがゼロ行。すべてのアプリはあなたかアシスタントが育てたもの。）*

メーカー投稿、HN 再ローンチ、ツイート#1 で一字一句そのまま繰り返す。

## 4. キーメッセージ（4つ、優先順）

PH の閲覧者が約10秒で覚えるのは**1つ**の考えだけ: **このアシスタントは買えない — 育てるものだ。** 残りはすべてその支柱。

**1. 育てるアシスタント。** — *デバイスは入り口、モデルはエンジン。アシスタントの価値はあなたについてどれだけ知っているかで決まる — そしてそれは買えない。蓄積するしかない。*
MulmoClaude をインストールすると、自分のマシンの上に空の畑ができる。話しかける: 必要なアプリを作り、学んだことをリンクされた wiki に整理し、すべての記録をプレーンファイルに残す。1週間であなたの文脈から答え、1か月でそれを横断する — *「どのクライアント会食を経費にすべき? 金曜はどこを予約する?」* に、数週間の時を隔てて自分で作った請求書トラッカーとレストランリストから答える。**これはどのベンダーにも売れない。価値があるのは彼らのソフトウェアではなく、あなたの蓄積された生活だから。これが中心 — PH のすべての面でこれを先頭に。**
- *記憶競争を一度だけ名指しする:* OpenAI、Anthropic、Meta、Google が揃ってアシスタントに記憶を接ぎ木している — 我々と同じことに気づいているからだ。その競争こそ、蓄積を彼らのサーバーではなく自分のディスクで起こすべき理由。
- *分断:* AI 時代の格差は、誰が最も賢いモデルを借りるかでは生まれない（みんな同じものを借りる）— 自分を本当に理解するアシスタントを育てたかどうかで生まれる。

**2. 会話で育つ — アプリも含めて。** — *「請求書トラッカーを作って」→ 本物のアプリが現れ、コードなし、その瞬間からアシスタントの一部。*
道具が要る? 頼めばいい。レストランリスト、請求書トラッカー、単語練習 — 一文からフィールド、計算済み合計、アクションボタンが現れ、ホストがテーブル / かんばん / カレンダーの3ビューを無料でレンダリングする。プログラマは AI にコードを書かせることを「バイブコーディング」と呼ぶ — ここではコードが見えることすらない。一人用アプリは AI がソフトウェアコストをほぼゼロにするまで経済的に不合理だった。MulmoClaude はその新しい経済を机の上に置く。
- *成長の一句 — 必ず添える:* アプリは使い捨ての成果物ではない（それはアプリビルダーのカテゴリ）。永続するアシスタントの新しい器官である。翌日そのアプリのデータを読み返す。レコードは相互参照し、派生値がアプリ横断で更新される（`value = shares × ticker.price` — 株価を1つ変えればポートフォリオ全体が再評価、同期コードなし）。
- *vs Lovable / Bolt / V0:* 彼らは生成して手離れする。ここでは生成が、永続するアシスタントの成長イベント。
- *vs Airtable / Notion / Retool:* あちらもノーコードだが、環境を設計するのはエンジニアで、中にエージェントは住んでいない。ここでは欲しいものを言葉にすれば、アシスタントが作り、かつ*運用する*。
- *エンジニア向けフレーミング:* 機能ごとにプラグインを書くのをやめる — スキーマを記述すれば、エージェントがその中で走る。

**3. 一つのアシスタント、すべての入り口 — 寝ている間も働く。** — *スマホもノート PC も同じアシスタントにログインする。中継サーバーはメッセージを運ぶだけ — データは家から出ない。*
同じ ID でスマホからログインすれば、PC に住むアシスタントに届く — 電車から、ソファから。中継サーバーは転送のみで何も保存しない: データも、アプリも、記憶も、誰のクラウドにも置かない。その間、畑はスケジュールで育つ: ソースを登録 → 朝のブリーフィングが待っている。定期的な義務を宣言 → 期日前に知らせ、次のサイクルに繰り越す。いずれはグラスやヘッドセット — 同じ家への入り口が増える。
- *vs デバイス常駐アシスタント（Siri、Alexa、グラス）:* 入り口はアシスタントではない。デバイスは自由に替えられ、アシスタントは丸ごと残る。
- *vs ワンショット・エージェント（Devin、Codex、素の Claude Code）:* 蓋を閉じたら止まる。こちらにはカレンダーがある。

**4. ロックイン不可能。** — *`~/mulmoclaude/` にプレーンな Markdown/JSON。git フレンドリー。オープンソース、MIT。離脱が自明に簡単 — だからこそ留まるのが安全。*
すべての会話、wiki ページ、アプリのスキーマ、レコードが、1つのフォルダのプレーンテキスト。`git push` がバックアップで、どのエディタでも開け、10年後も読める。オープンソースだから、どの会社も — 我々自身も含めて — 囲い込めない。これは機能ではなく、プロジェクトの存在理由（§0): これほど重要なアシスタントはレンタル品であってはならない。
- サンドボックス（Docker、自動検出）はここに畳み込む — Claude が見えるのはワークスペースだけ。SSH 鍵、`.env`、ホームディレクトリは不可視。「あなたのマシン、あなたのデータ」を*正直に保つ*仕組み。

### ビジュアルフック（デモの目玉。メッセージではない）

- **収穫。** 30日育てたアシスタントが、自分で作った2つのアプリから複合的な個人の質問に答える。我々が見せられる中で最も差別化された1枚 — アプリビルダーにもチャットボットにも再現できないショット。すべての冒頭に。
- **種。** *「請求書トラッカーを作って」* → 本物のアプリが数秒で実体化。見慣れた魔法（それでいい — これは入り口で、カットは必ず収穫→種の順。逆は禁止）。
- **入り口。** 路上のスマホからメッセージを送り、家のノート PC に答えが描かれるのを見せる — 同じアシスタント。キャプション: *「中継サーバーは何も保存しない」*。
- **株価1つ → ポートフォリオ全体が再評価**（`value = shares × ticker.price`）。アプリが本物である深さの証明。0:20 以降に。

### 10.5 アクティベーション問題 — これを解かねば当日を失う

依然として最大のギャップであり、育成フレーミングは賭け金を上げている: 製品の見返りは本質的に*累積的*なので、空っぽの初回体験はピッチ全体と自己矛盾する。`npx mulmoclaude` があっても、経路は Node + Claude CLI 認証 + Docker — 5〜10分、開発者限定、モバイルはゼロ。PH は即時の満足に報いる。**育った**アシスタントをゼロインストールで味わえる道がなければ、upvote は午前中に伸びて午後に失速する。

**選択肢3つ、優先順（A は v2 向けに再設計）:**

**A（強く推奨 — 未構築）: 30日育ったアシスタントのホステッド・ツアー。** 正典デモ・ワークスペース（§7）を読み取り専用でホスト: 来訪者はブラウザで約10の用意されたセッションを歩く — まず Day 30 の収穫の回答、次に Day 1 の種のリプレイ、ポートフォリオ再評価、実バックリンク付き wiki、スケジューラのキャッチアップ、スマホ往復のリプレイ。クリックスルー + キャンバスのリプレイ — 入力も認証も鍵も Docker も不要。v2 では v1 より強い: ツアー自体が「育てれば報われる」の証明であり、インストール先行の体験では初日に絶対に見せられないもの。予算 ~3 eng-days + ホスティング。T-10 までにサブドメイン + VM を確保。実現すればすべての CTA が *「ブラウザで育ったアシスタントを見学」* になる。

**B（最小限）: `--demo` リプレイモード。** `npx mulmoclaude --demo` が育成済みワークスペース + 収録セッションで約30秒起動 — Claude 認証も鍵も不要。離脱を半減。ただしモバイル/ゼロインストール層は失う。

**C（最終手段）: ランディングページのスクリプト化ウォークスルー。** Day 1 → Day 30 の物語をスクロールページで。インタラクティブ性はないが、流し読み層にタイムラプスの感覚は残せる。

**T-10（7月25日）までに決定: A にコミット。ダメなら B。C に触れる前に、まず日程を延期せよ。これらのどれもなしでローンチしない。**

## 13. 底流にある物語

HN、長文、「これはどこへ向かうのか」と問う人向け — ビジョン原典の議論と、それを支える脚:

**脚1 — アシスタントはデバイスでもモデルでもない。** グラスもスマホもスピーカーも入り口。GPT も Claude もエンジン。両方を剥ぎ取って残るもの — 本当のアシスタント — は、一人の人間についての蓄積された理解である: 会話、予定、連絡先、義務、メモ、道具。AI 大手はすべてこれに気づいた。だから「記憶」機能が一斉に現れている: 競争は最賢モデルのためではなく、最深の蓄積のためにある。そして、それほど価値があり、複利で効き、*一度育つと動かしにくい*資産こそ、他人の壁の内側で築いてはならない資産である。

**脚2 — 今日の AI エージェントはみなホームレスである。** 永続するファイルシステムなし、スケジュールなし、複利の記憶なし: 召喚され、働き、消える — それは関数呼び出しであってアシスタントではない。MulmoClaude はエージェントに家を与える: `~/mulmoclaude/`。会話から新しい道具が育つ工房（コレクション）、本棚（wiki）、書類棚（ドキュメント）、カレンダー（スケジューラ）、そして入り口（何も保持しない中継を通るスマホのブリッジ）。家があるから蓄積する。蓄積するから理解する。理解するから信頼と自律を獲得する。記憶 → 複利 → 信頼 → 委任。

その下の経済学: AI はソフトウェアのコストをゼロへ押し下げ、一人用アプリ — レストランリスト、請求書トラッカー、単語練習 — を計算機の歴史上初めて成立させた。ユーザーが欲しいものを語ること*が*仕様であり、アシスタントがビルダーであり実行環境である。（技術者向けには: アシスタントが書くスキーマはハーネス — エージェントがその中で動く、小さく読みやすい表面である。エンジニア向けの展開は [`docs/papers/dsl-as-harness.md`](../docs/papers/dsl-as-harness.md) と [`docs/papers/collections-architecture.md`](../docs/papers/collections-architecture.md) — コメント欄とブログの素材であって、PH の先頭には決して出さない。）

ローンチが成功したら、祝うのは製品ではない — AI 時代の分断を名指しすることになる: 最賢モデルを借りた者ではなく、自分を本当に理解するアシスタントを育てた者。そして人々に、その畑を手渡すのだ。
