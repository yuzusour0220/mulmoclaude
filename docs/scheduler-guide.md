# スケジューラー ガイド — 繰り返しタスクとカレンダー

## スケジューラーって何？

MulmoClaude のスケジューラーは、**AI に定期的な仕事を自動でやらせる仕組み**です。

たとえば：
- 毎朝9時にニュースを要約してもらう
- 1時間ごとにプロジェクトの進捗をチェックする
- 毎週月曜日にチームへのレポートを準備する

一度設定すれば、あなたが PC の前にいなくても AI が自動で動きます。PC をスリープから起こしたときには、その間にやるべきだった仕事を自動で取り戻します（キャッチアップ機能）。

さらに、**カレンダー**機能で予定やイベントを視覚的に管理できます。

---

## 2つの機能

スケジューラーには大きく分けて2つの機能があります：

### 1. カレンダー（予定の管理）

月表示・週表示・リスト表示で予定を見たり追加したりできます。

**できること：**
- 予定の追加（タイトル、日付、時刻、場所などを入力）
- 予定の編集・削除
- 月 / 週 / リスト表示の切り替え
- 日付なしの「未定」項目も管理

**表示方法：**
画面上部のプラグインランチャーから **Calendar（カレンダー）** を選ぶと、カレンダー画面が開きます。自動実行タスクは隣の **Actions（自動化）** タブに分かれています（旧「Schedule」メニューは 2026-04 以降 Calendar / Actions の 2 つに分割されました）。

**予定の追加方法：**
Claude に「明日の14時に会議を追加して」と頼むだけでOKです。もちろん UI から直接追加することもできます。

### 2. タスク（自動実行される仕事）

設定した時間になると、AI が自動でメッセージを処理します。

**できること：**
- 定期タスクの作成（「毎日9時に〇〇して」）
- タスクの一時停止 / 再開
- 手動で今すぐ実行
- 実行履歴の確認（いつ動いたか、成功したか）
- タスクの削除

---

## タスクの種類

### システムタスク（自動で動く、設定不要）

MulmoClaude が最初から持っている2つのタスクです。あなたが何もしなくても動きます。

| タスク | 何をする？ | 頻度 |
|---|---|---|
| **ジャーナル** | 最近のチャットを要約して日誌にまとめる | 1時間ごと |
| **チャットインデックス** | チャット履歴にタイトルと要約を付ける | 1時間ごと |

これらは **Actions** タブで確認できますが、変更や削除はできません（「System」バッジが付いています）。

### ユーザータスク（あなたが作る）

Claude に頼んで作成します。

**例：**

> 「毎朝9時にニュースサイトの最新記事を要約して」

と Claude に伝えると、Claude がタスクを自動で作成します。以降、毎朝9時になると Claude が自動的にそのタスクを実行します。

### スキルタスク（上級者向け）

スキルファイル（SKILL.md）に `schedule:` を書くと、そのスキルが自動的に定期実行されます。

```yaml
---
description: Slackの未読メッセージをまとめる
schedule: daily 08:00
---

Slack の未読チャンネルを確認して、重要なメッセージを箇条書きでまとめてください。
```

> **⚠️ 時刻はすべて UTC です。** `daily 08:00` は UTC 08:00 = **日本時間（JST）17:00** に実行されます（JST = UTC+9）。日本時間の朝8時に動かしたいなら、前日 UTC 23:00 にあたる `daily 23:00` を指定してください。

## タスクの作り方

### 方法1: Claude に頼む（おすすめ）

チャットで Claude に自然に伝えるだけです：

- 「毎日朝9時にやることリストを確認して」
- 「30分ごとにメールの要約を作って」
- 「毎週金曜の17時に週報を準備して」

Claude が理解して、適切なスケジュールのタスクを作成してくれます。

### 方法2: Actions タブから

1. 画面上部のプラグインランチャーから **Actions（自動化）** を選ぶ（旧「Schedule」→「Tasks」タブと同じ中身）
2. 登録済みタスクの一覧が表示される（各行にスケジュール・前回実行時刻・次回実行時刻）
3. 新規作成・編集は引き続き Claude 経由。UI は閲覧と手動実行のためのもの

---

## スケジュールの書き方

タスクには2種類のスケジュールがあります：

### 毎日決まった時刻

```
daily 09:00     → 毎日 9:00（UTC）
daily 18:30     → 毎日 18:30（UTC）
```

> **注意：** 時刻は UTC（世界標準時）です。日本時間の朝9時にしたい場合は `daily 00:00`（UTC 0:00 = JST 9:00）と設定します。

### 一定間隔

```
interval 30m    → 30分ごと
interval 2h     → 2時間ごと
interval 60s    → 60秒ごと（最短10秒）
```

## PC を閉じていたらどうなる？（キャッチアップ）

ノートPCを閉じて外出し、帰ってきて開いたとき — その間にスケジュールされていたタスクはどうなるでしょう？

MulmoClaude は**キャッチアップ**機能で、やり残しを自動で検知して処理します。

| ポリシー | 動作 | 使われる場面 |
|---|---|---|
| **スキップ** | やり残しを無視して次回から再開 | システムタスク（ジャーナル等） |
| **1回だけ実行** | 最新の1回分だけ実行 | ユーザータスク（デフォルト） |
| **全部実行** | やり残し全部を順番に実行 | 特別な設定が必要な場合 |

たとえば「毎日9時にニュースを要約」のタスクを設定していて、3日間 PC を閉じていた場合：
- **1回だけ実行（デフォルト）**: PC を開いた時点で最新の1回分だけ実行
- 3日分を全部やり直す必要はありません

## タスクの管理

### 一時停止と再開

タスクを削除せずに止めたいときは、**有効/無効**を切り替えられます。無効にしたタスクは時間が来ても実行されません。

### 今すぐ実行

スケジュールの時間まで待てないときは、**再生ボタン（▶）**をクリックすると即座に実行されます。

### 実行履歴

各タスクの実行結果を確認できます：
- いつ実行されたか
- 成功したか、エラーだったか
- どれくらい時間がかかったか
- エラーの場合、原因は何か

## カレンダーの使い方

### 表示の切り替え

カレンダータブには3つの表示があります：

- **月表示**: 1ヶ月のカレンダーグリッド。予定が多い日は「+3件」のように表示
- **週表示**: 7日間の予定を一覧
- **リスト表示**: 全予定を時系列で一覧

**◀ 今日 ▶** ボタンで前後の月/週に移動できます。

### 予定の追加

Claude に頼む方法が一番簡単です：

- 「6月20日の14時にチームミーティングを追加して。場所はZoom」
- 「来週の木曜に歯医者の予約を入れて」

予定には自由にプロパティ（場所、参加者、メモなど）を付けられます。

### 予定の編集・削除

- 予定をクリックすると編集画面が開きます
- リスト表示ではホバーで削除ボタン（×）が表示されます

---

## データの保存場所

スケジューラーのデータは MulmoClaude のワークスペース内に保存されます：

| ファイル | 中身 |
|---|---|
| `config/scheduler/items.json` | カレンダーの予定 |
| `config/scheduler/tasks.json` | ユーザーが作ったタスク |
| `config/scheduler/state.json` | タスクの実行状態（最後に動いた時刻など） |
| `data/scheduler/logs/` | 実行ログ（日別のJSONファイル） |

## よくある質問

### Q: タスクが動かない

- PC がスリープしていませんか？ MulmoClaude サーバーが起動している必要があります
- **Actions** タブでタスクが「有効」になっているか確認してください
- 実行履歴にエラーが記録されていないか確認してください

### Q: 日本時間で設定したい

時刻は UTC で設定します。日本時間（JST = UTC + 9時間）に変換してください：

| 日本時間 | UTC |
|---|---|
| 朝 6:00 | 21:00（前日） |
| 朝 9:00 | 00:00 |
| 昼 12:00 | 03:00 |
| 夕方 18:00 | 09:00 |
| 夜 21:00 | 12:00 |

### Q: タスクを全部止めたい

MulmoClaude サーバーを停止すれば、全てのタスクが止まります。個別に止めたい場合は **Actions** タブで無効にしてください。

### Q: カレンダーの予定は自動で何かする？

いいえ。カレンダーの予定は表示用です。自動実行させたい場合は「タスク」として作成してください。
